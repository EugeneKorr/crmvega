# 🚩 Feature Flag для редизайна — Как использовать

## Что это?

**Feature flag** (`newDesign`) позволяет безопасно тестировать новый дизайн Ant Design v6 без влияния на основной код:

- ✅ Старый дизайн работает по умолчанию в production
- ✅ Новый дизайн доступен для разработки и тестирования  
- ✅ Легко переключаться между версиями
- ✅ Можно мерджить в main без риска

---

## 🎯 Как включить новый дизайн

### **Способ 1: Query параметр (один раз)**
```
http://localhost:3000?newDesign=1
```
Новый дизайн включится **только в этом сеансе**.

### **Способ 2: Кнопка в приложении (сохраняется)**
1. Откройте приложение
2. Найдите кнопку **🎨 (палитра)** в шапке (рядом с уведомлениями и профилем)
3. Кликните — дизайн переключится
4. Выбор сохранится в `localStorage` и будет восстанавливаться при входе

### **Способ 3: DevTools Console**
```javascript
localStorage.setItem('crm_use_new_design', 'true');
location.reload();
```

---

## 📋 Как это работает внутри

```
┌─ DesignProvider (App.tsx)
│  └─ Читает query param ?newDesign=1
│  └─ Читает localStorage 'crm_use_new_design'
│  └─ Сохраняет флаг в DesignContext
│
├─ AppContent
│  └─ Читает useNewDesign из контекста
│  └─ Применяет новую тему ИЛИ дефолтную
│
└─ DesignToggle (компонент в шапке)
   └─ Показывает статус (✅ зелёный = новый, серый = старый)
   └─ Переключает флаг и сохраняет в localStorage
```

---

## 🔄 Порядок разработки

```
1. Работаете на feature/redesign-v6
2. Запускаете ?newDesign=1
3. Видите новый дизайн
4. Делаете изменения (например, в SuggestionBar.tsx)
5. Guardian проверяет что функциональность не сломалась
6. Мерджите в main (старый дизайн по умолчанию)
7. Позже, когда всё готово, удаляете флаг
```

## ❌ Отключить новый дизайн

### **Способ 1: Убрать из URL**
```
http://localhost:3000  (без ?newDesign=1)
```

### **Способ 2: Кликнуть на кнопку 🎨 ещё раз**

### **Способ 3: Console**
```javascript
localStorage.removeItem('crm_use_new_design');
location.reload();
```

## 🧪 Тестирование

**Убедитесь что:**
- [ ] Старый дизайн работает по умолчанию (без флага)
- [ ] Новый дизайн включается через `?newDesign=1`
- [ ] Кнопка 🎨 переключает дизайны
- [ ] Выбор сохраняется после перезагрузки
- [ ] Функциональность одинаковая в обоих дизайнах
- [ ] Консоль браузера чистая (нет ошибок)

## 🎨 Структура файлов

```
frontend/src/
├── contexts/
│   └── DesignContext.tsx       ← управление флагом
├── components/
│   ├── DesignToggle.tsx        ← кнопка в шапке
│   └── MainLayout.tsx          ← где используется DesignToggle
├── theme/
│   ├── index.ts                ← конфиг новой темы
│   └── tokens.json             ← дизайн-токены из Figma
└── App.tsx                     ← где применяется тема
```

---

## 📝 Когда удалить флаг?

Когда редизайн **полностью завершён** и протестирован:

```bash
# 1. Удалить DesignContext из App.tsx (применить тему всегда)
# 2. Удалить DesignToggle из MainLayout
# 3. Удалить DesignContext.tsx и DesignToggle.tsx файлы
# 4. Обновить импорты
# 5. Создать финальный коммит
```
